## 要解决的问题
1. 系统每次只产生一条日志，如果每条日志都调用log sdk发送，对网络IO的使用效率很低。
2. 少量日志每次都发送，这样就会有大量的tcp短链接，很容易耗尽系统端口。

log producer解决上面问题的方法是会将日志merge到一定数量才真正发送，具体数量由根据用户配置的参数决定。
## log producer功能
1. 提供异步的发送接口，线程安全。
2. 可以添加多个project的配置。
3. 用于发送的网络IO线程数量可以配置。
4. merge成的包的日志数量以及大小都可以配置。
5. 内存使用可控。
